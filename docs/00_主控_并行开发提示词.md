# 用户端页面并行开发 · 主控提示词

## 任务概述

为 CareerAI 职业规划智能体开发用户端移动端页面。使用 **subagent 并行模式**同时开发以下8组页面，每组独立运行，最后汇总。

**技术栈：** React + TypeScript + Vite，React Router v6，纯 CSS（无 Tailwind），手机容器模拟（宽 200~220px）

**仓库：** https://github.com/fengfangwen061026/career-planning-app

**目录约定：**
- 所有页面写入 `frontend/src/pages/mobile/`
- 全局样式变量写入 `frontend/src/styles/mobile-tokens.css`（仅由 Group A 创建，其他复用）
- 全局组件写入 `frontend/src/components/mobile/`

---

## 并行任务分配

使用 subagents 同时执行以下8个任务：

```
Task A → 01_GroupA_引导登录.md
Task B → 02_GroupB_上传解析.md
Task C → 03_GroupC_学生画像.md
Task D → 04_GroupD_对话补全.md
Task E → 05_GroupE_岗位探索.md
Task F → 06_GroupF_匹配详情.md
Task G → 07_GroupG_报告页.md
Task H → 08_GroupH_空状态反馈.md
```

**执行顺序：**
1. 先执行 Task A（建立 mobile-tokens.css 和 MobileShell 容器组件）
2. Tasks B~H 在 Task A 完成后并行执行
3. 所有 Task 完成后，更新 `frontend/src/App.tsx` 汇总路由

## 完成后汇总步骤

所有 Task 完成后，在 `frontend/src/App.tsx` 配置路由：

```tsx
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom'
// 引入所有页面组件...

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Navigate to="/onboarding" replace />} />
        <Route path="/onboarding" element={<OnboardingFlow />} />
        <Route path="/upload" element={<UploadPage />} />
        <Route path="/parsing" element={<ParsingPage />} />
        <Route path="/profile" element={<ProfilePage />} />
        <Route path="/chat-fill" element={<ChatFillPage />} />
        <Route path="/explore" element={<ExplorePage />} />
        <Route path="/match/:jobId" element={<MatchDetailPage />} />
        <Route path="/report" element={<ReportPage />} />
      </Routes>
    </BrowserRouter>
  )
}
```
